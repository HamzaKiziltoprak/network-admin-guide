1. IP Address Conflict (Network Layer - Layer 3)

Problem Definition:

An IP address conflict occurs when two or more devices on the same network are configured with the same IP address. This leads to unexpected behavior, communication errors, and loss of connectivity for affected devices.

Potential Causes:

  • Manual static IP assignment errors by administrators.

    When administrators manually assign static IP addresses, they may accidentally assign the same IP to multiple devices, leading to conflicts.

  • Improper or overlapping DHCP address pool settings.

    DHCP servers may be configured with overlapping address pools, causing multiple devices to receive the same IP address.

  • Device cloning or OS imaging without updating IP settings.

    Cloning devices or using OS images without reconfiguring network settings can result in duplicate IP addresses across devices.

  • Rejoining devices attempting to use old IPs.

    Devices that were previously disconnected from the network may attempt to reuse their old IP addresses, which might already be assigned to another device.

Solution Steps:

  1. Scan the network using tools like arp -a or network mapping to detect duplicate IPs.
  2. Identify devices sharing the same IP manually or through simulation.
  3. Reassign a unique IP to one of the conflicting devices (e.g., change from 192.168.1.10 to 192.168.1.12).
  4. Check and correct DHCP server configuration to prevent overlapping assignments.
    1. Click on the router in Packet Tracer and switch to the CLI tab.
    2. Check the DHCP pool configuration by running the following command: 
      show running-config | include dhcp
    3. If there is an IP range conflict, correct the DHCP settings: 
      
            configure terminal
            ip dhcp excluded-address 192.168.1.10 192.168.1.20
            ip dhcp pool MYPOOL
            network 192.168.1.0 255.255.255.0
            default-router 192.168.1.1
            end
    4. Restart the DHCP service to apply the changes: 
      clear ip dhcp binding *
  5. Ping test all IPs to verify uniqueness and restore connectivity.
  6. Document IP usage and consider using IP address management tools.

2. Inter-Vlan Routing Issue

Problem Definition:

Inter-VLAN routing is the process of forwarding traffic between different VLANs (Virtual Local Area Networks) on a network. This is typically done using a Layer 3 device, such as a router or a Layer 3 switch. Without proper inter-VLAN routing, devices in different VLANs cannot communicate with each other.

Potential Causes:

  • Missing or misconfigured VLAN interfaces on the router or Layer 3 switch.
  • Incorrect IP addressing or subnetting for VLANs.
  • Switchport configurations not set to trunk mode for inter-VLAN communication.

    no encapsulation dot1Q 30

    interface gigabitEthernet0/1.30
    no encapsulation dot1Q 10
    encapsulation dot1Q 30
    ip address 172.17.30.1 255.255.255.0
    exit
    interface gigabitEthernet0/1.10
    encapsulation dot1Q 10
    ip address 172.17.10.1 255.255.255.0
    no shutdown
    exit

3. DHCP Server Issues

Problem Definition:

Devices fail to obtain IP addresses automatically due to DHCP server misconfigurations or failures. This can result in devices being unable to communicate on the network, leading to connectivity issues and disruptions in network services. Without a valid IP address, devices cannot access shared resources, connect to the internet, or communicate with other devices on the network.

Potential Causes:

  • DHCP server is not enabled or running.

    The DHCP server may be turned off or not functioning, preventing devices from obtaining IP addresses.

  • IP address pool is exhausted.

    The DHCP server has run out of available IP addresses to assign to devices.

  • Network devices are not configured to use DHCP.

    Devices may be set to use static IPs instead of obtaining them automatically from the DHCP server.

Simulation File:

Download Packet Tracer Simulation

DHCP fail message on pc 8

Solution Steps:

  1. Verify that the DHCP server is enabled and running
    Open the CLI on the router and configure the interface connected to the switch: 
    Router> enable
Router# configure terminal
Router(config)# interface GigabitEthernet0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# exit
    Then configure the DHCP service on the router: 
    Router(config)# ip dhcp pool LAN
Router(dhcp-config)# network 192.168.1.0 255.255.255.0
Router(dhcp-config)# default-router 192.168.1.1
Router(dhcp-config)# dns-server 8.8.8.8
Router(dhcp-config)# exit
  2. Check the IP address pool and expand it if necessary
    If one or more PCs do not receive an IP address (e.g., PC9), the address pool may be too small. Check for excluded addresses: 
    Router# show running-config
    If the range is too narrow, reduce the excluded IP range like this: 
    Router(config)# no ip dhcp excluded-address 192.168.1.1 192.168.1.191
Router(config)# ip dhcp excluded-address 192.168.1.1 192.168.1.189
    This change increases available IPs in the pool for more clients.
  3. Ensure devices are set to obtain IP addresses automatically
    For each PC:
    • Click on the PC.
    • Go to the Desktop tab.
    • Open IP Configuration.
    • Select the DHCP option (instead of Static).
    After selecting DHCP, the PC should receive an IP like 192.168.1.190 or higher (depending on the available range).

4. DNS Resolution Problems

Problem Definition:

Devices cannot resolve domain names to IP addresses, leading to connectivity issues.

Potential Causes:

  • Incorrect DNS server configuration on devices.
  • DNS server is down or unreachable.
  • DNS cache corruption.

Solution Steps:

  1. Verify DNS server settings on devices.
  2. Check the status of the DNS server and restart if necessary.
  3. Clear DNS cache using ipconfig /flushdns.

5. Firewall Misconfiguration

Problem Definition:

Firewall rules block legitimate traffic, causing connectivity issues.

Potential Causes:

  • Overly restrictive firewall rules.
  • Incorrect port or protocol settings.
  • Firewall software or hardware malfunction.

Solution Steps:

  1. Review and update firewall rules to allow necessary traffic.
  2. Verify port and protocol configurations.
  3. Restart or update the firewall software or hardware.

6. NAT Configuration Issues

Problem Definition:

Devices on a private network cannot access the internet due to NAT misconfigurations.

Potential Causes:

  • Incorrect NAT rules or settings.
  • Overlapping IP address ranges.
  • Insufficient NAT resources on the router.

Solution Steps:

  1. Verify and correct NAT rules on the router.
  2. Ensure IP address ranges do not overlap.
  3. Upgrade router resources if necessary.

7. Wi-Fi Connectivity Problems

Problem Definition:

Devices experience slow or intermittent Wi-Fi connections.

Potential Causes:

  • Signal interference from other devices.
  • Incorrect Wi-Fi settings or credentials.
  • Overloaded access points.

Solution Steps:

  1. Reduce interference by changing the Wi-Fi channel.
  2. Verify Wi-Fi settings and credentials on devices.
  3. Add more access points to distribute the load.

8. Switch Loop Issues

Problem Definition:

Network experiences broadcast storms due to switch loops.

Potential Causes:

  • Redundant connections between switches without STP (Spanning Tree Protocol).
  • Misconfigured switch ports.

Solution Steps:

  1. Enable STP on all switches.
  2. Remove redundant connections or configure them properly.

9. MTU Configuration Problems

Problem Definition:

Large packets are dropped due to incorrect MTU settings.

Potential Causes:

  • MTU size mismatch between devices.
  • Fragmentation issues.

Solution Steps:

  1. Set the MTU size to match across all devices.
  2. Enable packet fragmentation if necessary.

10. QoS Configuration Issues

Problem Definition:

Critical traffic experiences delays due to lack of QoS policies.

Potential Causes:

  • No QoS policies configured.
  • Incorrect prioritization of traffic.

Solution Steps:

  1. Implement QoS policies to prioritize critical traffic.
  2. Verify and adjust traffic prioritization settings.
